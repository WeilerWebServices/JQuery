TJ VanToll

jQuery UI Widgets vs. HTML5

HTML5 has brought native capabilities to the browser that were previously only available through JavaScript libraries such as jQuery UI. Examples include <input type="date"> vs. datepicker, <input type="range"> vs. slider, <input type="number"> vs. spinner, and <datalist> vs. autocomplete.

The overlapping functionality leaves web developers to choose between native options and JavaScript based widgets. In this talk we'll discuss the pros and cons of each approach to help you decide which makes sense for your application.

Patrick Camacho

Beyond DOM Manipulation: Building Stateful Modules with Promises and Events

State management can be a difficult problem to solve when building web applications with Javascript. Web apps are inherently stateful, as opposed to traditional web sites.

I'm going to outline an approach to state management that uses modules to keep code clean and organized, but also relies on promises and events to allow safe, asynchronous communication between modules and within the module itself.

This approach saves time in development, reduces code bloat, prevents callback chains, and safely shares information between modules using the observer pattern.

Howard Abrams

A/B Testing: How To with jQuery

We've all attended meetings where the designer and the copy editor can't agree on a critical part of your web application. They each argue their case before the boss, who ultimately chooses. However, since no customers attended the meeting, how would the boss know the best choice?

The solution? Perform an experiment on some of your customers as test subjects. Show one group *Choice A*, and another group *Choice B*. Easy, right? Just remember, the grouping needs to be well distributed as well as random, and you'll need to measure and report the results.

Multivariate Testing (also called A/B or Split Testing) lets you determine the best "choice" by showing a choice to a subset of your customers and measuring the results. Of course, you've already heard about it since Google has been doing these sorts of experiments for years, but how to begin? Does this require backend support or can this be done completely in jQuery?

If your company is anything like mine, you'll have many concurrent tests, so make sure not to pick on some customers too much, or you'll skew the results.

I'll show you the tools you'll need and code you'll write, with plenty of examples and a jQuery A/B Testing plugin. I'll share tips and tricks from the trenches to improving your customer experiments, and making your decisions more scientific.

BUGWILD, or, RESOLVED INCOMPLETE WORKSFORME WONTFIX

There is a great deal of advice about how to use a growing list of powerful tools and techniques to build robust JavaScript applications. That's not what this talk is about, though, because none of it will save you: in spite of your tests and your best practices, your app *will* fail in the hands of your users and it will do so in ways that surprise you.

I will instead discuss what to do when it all goes sideways. I'll show you how to detect failures in the first place, what to tell your users and when, and how to gather the forensic data available in the browser. jQuery is a Swiss Army Knife (and that's OK!)

As jQuery has grown to become one of the most widely used software libraries on the planet, so too has consternation and resentment about how it should be used and by whom.
